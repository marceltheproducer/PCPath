# UNC Input Support Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Accept `\\server\share\rest` UNC paths as input in all five converter surfaces (Windows PS ×2, Mac shell, Mac Swift, web JS), mirroring the existing `smb://` handling: drop the host, share name = volume name. Then run the full Windows release (installer 2.3 → 2.4, web v1.4.0 → v1.5.0, sync all 4 locations, portal bump).

**Architecture:** UNC is a normalize-early input shape. Each surface converts `\\host\share\rest` → `/Volumes/share/rest` at the same point where it already handles `smb://`; downstream logic (volume→drive mapping, fallback discovery, `?(NAME)` placeholder, suffix stripping) is untouched.

**Tech Stack:** PowerShell 5.1, bash, Swift (Foundation-only), vanilla JS in a single HTML file, NSIS via `sync.ps1`.

**Spec:** `docs/superpowers/specs/2026-07-15-unc-input-design.md`

## Global Constraints

- **Edge rules (identical in every surface):** convert only backslash UNC `\\host\share[...]` where host is not `?` or `.`; `\\host` alone, `\\?\...`, `\\.\...`, and `//host/share` pass through unchanged; no URL-decoding; `\Volumes`/`\\Volumes` typo-normalization keeps precedence over UNC handling.
- **Never** `git reset --hard` / restore / clean. The repo has pre-existing dirty files (`kandji/build_pkg.sh`, `kandji/uninstall_mdm.sh`, `pcpath_common.sh`, `remote_install.sh`, `Convert to Mac Path.workflow/Contents/Info.plist`) — leave them untouched; stage only files this plan names.
- PS 5.1 gotchas: no `&&`/`||`; never `2>&1` on native `git`; `Get-Content`/`Set-Content` on repo text files needs `-Encoding UTF8`.
- Repo root: `v:\General Dev\Design\PCPath`. All relative paths below are from there. "LF will be replaced by CRLF" git warnings are normal.
- Test commands (run from repo root): `powershell -NoProfile -File tests\run_windows.ps1` · `bash tests/run_shell.sh` (Git Bash) · `node tests/run_web.mjs`. Each prints `ok`/`FAIL` lines and exits nonzero on failure. Swift tests cannot run on this machine (no swiftc) — they execute at the next build-Mac session.
- Commit messages end with: `Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>`
